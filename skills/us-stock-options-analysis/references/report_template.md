# V4 报告模板（reportMarkdown 标题树）

`reportMarkdown` 必须按下列 **H1/H2** 标题顺序保留全部十二章标题（供 reportSections 树结构），但**每章正文只写 1-3 句关键要点 + 关键数字，务必精简、不写长表格、不逐章长篇**——详细数据放结构化 `sections`/`options`。**正文越短越能在 KimiClaw 单任务预算内一次吐完，绝不能因报告过长被 terminated。** 九维评分明细放结构化 `sections`，正文一句话给总分与解读即可。

## 标题树（固定）

```markdown
# {公司名}（{TICKER}）美股基本面与期权分析报告

## 结论摘要

## 0. 基本信息与数据口径

## 一、业务组成与结构

## 二、竞争关系对比分析

## 三、最近增速与财务表现

## 四、未来增速与财务预期

## 五、股价预估与估值分析

## 六、期权市场情绪分析

## 七、期权关键价位分析

## 八、期权-基本面交叉验证

## 九、核心风险因素

## 十、投资结论与建议

## 十一、组合配置建议

## 十二、九维评分
```

## 章节最低要求

| 章节 | 必须包含 |
|------|----------|
| 结论摘要 | 一句话核心逻辑、综合评分、建议操作、期权时点结论（🟢/🟡/🔴） |
| 0. 基本信息 | 日期、现价、市值、财报/大事件、数据来源、`options.dataAvailable` |
| 一~五 | v4 基本面表格 + 关键判断（见 `framework.md`） |
| 六~八 | 有近月/远月两到期日对比表；无期权链时整章写「数据不可用」+ 不编造数字 |
| 九 | 风险表含概率/影响/监控指标/期权信号列 |
| 十 | 3/6/12 个月目标价、仓位、买卖区间、止损、监控指标 |
| 十一 | 组合相关性、仓位建议、再平衡规则；无持仓则给通用配置框架 |
| 十二 | 九维评分表 + 加权总分 + 评分解读 |

## 期权不可用时的写法（六~八章）

不得跳过章节。统一写法：

```markdown
## 六、期权市场情绪分析

> 结构化期权链不可用（`options.dataAvailable=false`）。本节不输出 P/C、IV、Max Pain 等数值。

- 近月到期日：待验证
- 远月到期日：待验证
- 情绪判断：不可判断
- 期权时点信号：🟡 观望（等待期权链数据）
```

七、八章同理，写清「不可判断」，禁止编造。

## 逻辑与风格

- 每节结论用「因为 A，所以 B，建议 C」
- 数字必须标注来源与日期；缺数据写 `待验证`
- 期权信号：🟢 买入时机 / 🟡 观望 / 🔴 减仓或回避
- 风险提示独立成章，不回避负面因素
